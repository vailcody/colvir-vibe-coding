# Unit Tests

Когда нужно написать unit-тесты для компонента, функции или API route:

## Стек
- Vitest (НЕ Jest)
- @testing-library/react для компонентов
- Файлы тестов рядом с исходником: `*.test.ts` / `*.test.tsx`

## Процедура

1. Прочитай исходный файл и определи все экспортируемые функции/компоненты
2. Для каждой функции определи:
   - Входные параметры и их типы
   - Ожидаемый результат
   - Граничные случаи (пустые данные, null, невалидный ввод)
   - Ошибочные сценарии
3. Создай файл теста рядом с исходником
4. Напиши тесты по структуре:

```typescript
import { describe, it, expect, vi } from 'vitest'

describe('имяФункции', () => {
  it('должен вернуть результат при валидных данных', () => {
    // arrange
    // act
    // assert
  })

  it('должен выбросить ошибку при невалидном вводе', () => {
    expect(() => fn(null)).toThrow()
  })

  it('должен обработать пустые данные', () => {
    expect(fn([])).toEqual([])
  })
})
```

## Правила
- Каждый тест проверяет одну вещь
- Названия тестов на русском (для наглядности на воркшопе)
- Внешние зависимости мокаются через `vi.mock()`
- API-вызовы мокаются через `vi.fn()`, не делать реальные запросы
- Не тестируй внутреннюю реализацию — тестируй поведение
- Минимум 3 теста на функцию: happy path, edge case, error case

## Для API routes
- Используй fetch для вызова route handler напрямую
- Проверяй: статус-код, формат ответа `{ success, data, error }`, валидацию входных данных
