## Текущие режимы Cursor (v2.6)

| Режим | Шоткат | Что делает | Когда использовать |
|-------|--------|-----------|-------------------|
| **Ask** | ⌘L | Read-only: вопросы, объяснения, impact analysis | Перед сложными изменениями, исследование кода |
| **Plan** | Shift+Tab | Исследует кодовую базу → генерирует план с Mermaid-диаграммами | Фичи на 10+ файлов, сложная декомпозиция |
| **Agent** | ⌘I | Основной режим. Автономно: читает, пишет, запускает терминал, итерирует | Реализация фич, рефакторинг, генерация тестов |
| **Debug** | Через палитру | Инструментирует логи, анализирует root cause | Непонятные ошибки, runtime-проблемы |
| **Edit** | ⌘K | Inline-правки выделенного кода | Точечные изменения, рефакторинг одной функции |

### Золотой воркфлоу
```
Ask (исследование) → Plan (декомпозиция) → Agent (реализация) → Ask (проверка)
```

**Важно:** YOLO mode — устаревший термин. Agent по умолчанию автономно запускает команды, это стандартное поведение.

## Контекст и @-mentions

| @ | Что делает | Совет |
|---|-----------|-------|
| @filename | Конкретный файл | Начинай с этого — узкий контекст = лучше результат |
| @folder | Вся папка | Работа с модулем целиком |
| @codebase | Поиск по всему индексу | Для широких вопросов, не для каждого запроса |
| @docs | Документация библиотек | Актуальная документация вместо устаревших знаний модели |
| @web | Интернет-поиск | Свежие решения, changelog |
| @git | История коммитов | Понять зачем что-то менялось |

**Правило**: узкий контекст → расширяй только если нужно. @codebase на каждый запрос сжирает токены.

## .cursor/rules (правила проекта)

```
.cursor/
  rules/
    backend.mdc      # Правила для бэкенда (globs: src/api/**)
    frontend.mdc     # Правила для фронта (globs: src/components/**)
    security.mdc     # Правила безопасности (alwaysApply: true)
```

- **alwaysApply: true** — применяется к каждому чату
- **globs** — применяется только к файлам по маске
- **description** — когда Cursor должен подтянуть правило автоматически

### Пример для Колвира:
```markdown
---
alwaysApply: true
---
# Правила безопасности

## Критично
- Все SQL через параметризованные запросы (NO string concatenation)
- Валидация входных данных через Zod
- Не хранить секреты в коде — использовать env переменные
- Логирование без PII (персональных данных)

## Стек
- TypeScript strict mode
- PostgreSQL через ORM
- Обработка ошибок через try/catch с типизацией
```

## Стратегии работы с ошибками (актуальные)

1. **Правило 8 сообщений** — если за 8 итераций агент не решил проблему, начни новый чат
2. **Git Checkpoint Workflow** — git stash / commit перед каждым крупным запросом
3. **Branch-per-task** — отдельная ветка на каждую AI-задачу
4. **Ask перед Agent** — сначала исследуй, потом реализуй
5. **Debug mode** — не пытайся вручную, используй встроенный дебаггер
6. **Scope control** — не давай агенту задачу на >15 файлов, декомпозируй

## Best Practices для разработчиков Колвира

1. **Privacy Mode** — включить обязательно (банковский софт)
2. **Rules** — сразу настроить .cursor/rules с правилами безопасности
3. **Начинать с Ask** — не бросаться в Agent на незнакомом коде
4. **MCP PostgreSQL** — подключить, чтобы агент проверял SQL
5. **Automation Security Review** — настроить для автоматического ревью PR
6. **Subagents** — для больших рефакторингов и миграций
